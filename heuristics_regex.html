Як це виконується в Python

КрокЩо робить кодДе в коді
1. Parsed documents Processor завантажує документи тендера, парсить текст і передає список ParsedDocument(document, text) в analyzer. processor.py → process_tender_payload()
2. RULES loop TenderAnalyzer.analyze() проходить по кожному документу і кожному Rule з RULES. analyzer.py → TenderAnalyzer.analyze()
3. Regex match Для кожного правила виконується rule.pattern.finditer(text). Якщо є match, навколо нього береться evidence snippet. rule.pattern.finditer(), evidence_window()
4. Skip guards should_skip_match(category, quote) перевіряє safe contexts: наприклад АВК у актах виконаних робіт або ISO в закупівлі послуг сертифікації. should_skip_match()
5. Issue object Якщо match не відкинуто, створюється Issue: category, title, severity, evidence_quote, explanation, suggested_rewrite, document title/id. Issue(...)
6. Extra heuristics Після regex rules додаються computed евристики: відсутні умови оплати/поставки/договору, щільні технічні параметри, проблеми парсингу. missing_context_issues(), technical_precision_issues(), parsing_issues()
7. Score Issues дедуплікуються і штрафують overall/subscores: висока = 14, середня = 8, низька = 4. dedupe_issues(), score_issues()
